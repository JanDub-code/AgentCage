# AGENTS.md

## Builds & development

```bash
cargo build                # debug build
cargo build --release      # release build
cargo run -- <args>        # run via cargo
```

Tests live in `src/lib.rs`. No lint/format config.

## Runtime dependency: Podman (not Docker)

The CLI probes for `podman` at runtime. Docker is explicitly rejected. Rootless Podman is required (`podman info --format {{.Host.Security.Rootless}}` must return `true`).

## Project structure

```
src/main.rs    — entry point (3 lines, calls agentcage::run)
src/lib.rs     — all CLI logic: parsing, podman commands, output, completions
src/agent.rs   — agent name → command mapping (codex → codex, etc.)
Dockerfile     — Ubuntu 24.04 image with codex, claude, opencode installed via npm
Cargo.toml     — single dep: libc (for isatty)
```

No config files. No serde. No external policy engine.

## Architecture principles

This was aggressively stripped down from a larger policy/workbench/audit system. **Do not add back:**
- policy engine or policy.json
- workspace diffs, staging, or apply
- secret scanning
- session history or audit logs
- desktop/Tauri UI
- multi-agent dashboards

The entire value is: mount one directory into a disposable container and run a single agent.

## Key paths inside the container

| Host                          | Container              |
|-------------------------------|------------------------|
| `$PWD`                        | `/workspace/project`   |
| (nothing — tmpfs)             | `/workspace/session-home` |
| (nothing — not mounted)       | host `$HOME`, `.ssh`, `.aws`, etc. |

## Adding a new agent

Three places must be updated together:

1. `src/agent.rs` — add to `SUPPORTED_AGENTS` array
2. `src/lib.rs` — add to the `match cmd.as_str()` arm in `parse()`
3. `src/lib.rs` — add to `print_help()`, `print_completions_bash()`, `print_completions_zsh()`, `print_completions_fish()`
4. `Dockerfile` — the agent binary must exist in the image

## Image workflow

```bash
ac         # checks podman, builds image if missing, creates .agentcage/
ac codex   # runs `podman run --rm ... agentcage/agent-tools:local codex`
```

The image tag is hardcoded as `agentcage/agent-tools:local` (`IMAGE` constant in lib.rs). `ac` with no subcommand builds it from the `Dockerfile` embedded in the compiled binary, so installed releases can initialize projects from any directory. `ac init` remains an explicit alias.

## Shell completions

Generated by the binary itself, not via clap:

```bash
source <(ac completions bash)
source <(ac completions zsh)
ac completions fish | source
```

The completions strings are hand-maintained in `src/lib.rs`. If you add or rename a command, update all three completion functions.
